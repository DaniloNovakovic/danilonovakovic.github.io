# Proposal Template

## Title

Name the design direction in one memorable phrase.

## Pillars

List 3-5 principles that make decisions easier. Each pillar should imply a tradeoff.

## Overworld Loop

Describe the player's minute-to-minute loop:

1. What draws the player forward?
2. How do they discover mini-games or secrets?
3. What changes when they return from a mini-game?
4. How does the world stay compact?

## Progression

Define the durable unlocks:

- movement upgrades
- knowledge gates
- mini-game completion markers
- items or keepsakes
- final route requirements

Keep the progression model small enough for bridge-owned state.

## Mini-Game Concepts

For each concept:

- **Theme**
- **Research inspiration**
- **Core verb**
- **Session length**
- **Win/loss or reset condition**
- **Mobile control profile**
- **Reward**
- **Out of scope**
- **Why it is fun**

## Mobile Controls

State the touch model for the overworld and for every mini-game. Avoid keyboard-only secrets and tiny active-play controls.

## Implementation Scope

Explain the first vertical slices and the expected folder shape. Name anything that should stay local to one scene.

## Risks

List the largest design and architecture risks, plus mitigation.

## Fun Thesis

End with the one emotional reason this version should exist.
